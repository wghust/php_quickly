###这个是一个文件布局框架
##目的：更快的布局
---
```
    public文件夹用于共用的，用户能够有权限访问
    sys文件夹用于系统调用。

    public文件夹下包括三部分
    第一部分是静态文件，类似index.html,而index.php用于跳转路由，
    如果只是静态文件就不用了。
    第二部分是assets文件夹，用于放置css，js，images等文件
    第三部分是php等后台文件放置地址。用于静态文件与后台的ajax交互。

    sys文件夹下包括三部分
      class文件夹，用于与数据库交互
      config文件夹，用于配置数据库连接参数
      core文件夹，用于动态调用class文件夹下的类文件。
```

```
    如果不访问数据库，只需要在public撰写代码就可以了
    需要的话，得配置config下的文件夹。
```
